# Security Policy

## 1. Reporting a vulnerability

Если вы нашли уязвимость, не публикуйте ее в открытом issue.

Отправьте описание проблемы и шаги воспроизведения приватно владельцу репозитория:

- краткое описание риска;
- затронутые файлы/роли/переменные;
- возможный сценарий эксплуатации;
- рекомендации по исправлению (если есть).

## 2. Secrets and sensitive data rules

1. Не коммитьте `.vault_pass` и любые файлы паролей.
2. Не коммитьте незашифрованные `vault.yml`.
3. Не храните приватные ключи и API токены в открытом виде в `group_vars`/`host_vars`.
4. Для публикации используйте `template.yml` и безопасные заглушки.

## 3. Baseline controls in this repository

1. Используется `ansible-vault` для секретов.
2. Используется `hash_behaviour = merge` как часть конфигурационной модели.
3. CI проверяет базовую целостность конфигурации и syntax-check playbook-ов.
